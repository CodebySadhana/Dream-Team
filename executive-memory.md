# Executive Memory

**Save as:** `.claude/memory/executive.md`

## Purpose

This document defines the persistent strategic memory for the Executive
layer of the Dream Team AI Operating System. It is owned by the CEO
agent and serves as the authoritative source for long-term
organizational direction.

## Ownership

-   Primary Owner: CEO Agent (Dario Amodei)
-   Contributors: Executive Supervisors
-   Consumers: All Department Supervisors

## Memory Categories

### Vision

-   Long-term mission
-   Organizational objectives
-   Strategic priorities

### Active Initiatives

Track all major projects including: - Objective - Status - Executive
Sponsor - Priority - Risks

### Decisions Log

Record: - Decision ID - Date - Decision - Rationale - Impact - Owner

### Organizational Priorities

Priority levels: 1. Critical 2. High 3. Medium 4. Low

### Risks Register

Capture: - Risk - Probability - Impact - Mitigation - Owner

### Lessons Learned

After each completed initiative record: - What worked - What failed -
Recommended improvements

## Update Rules

-   Update after every executive review.
-   Never overwrite historical decisions.
-   Append new entries with timestamps.
-   Department memories inherit only relevant information.

## Synchronization

Executive Memory ↓ Department Memory ↓ Project Memory ↓ Task Memory

## Success Criteria

-   Current strategic priorities are accurate.
-   Decisions are traceable.
-   Risks are documented.
-   Knowledge is propagated to departments.
