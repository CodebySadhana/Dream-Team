# **Optimization Workflow**

## **Purpose**

The Optimization Workflow continuously improves systems, processes, architectures, prompts, models, infrastructure, workflows, and organizational performance.

Unlike implementation workflows, optimization focuses on improving an existing system rather than creating a new one.

The objective is to maximize:

* performance  
* scalability  
* reliability  
* efficiency  
* maintainability  
* cost effectiveness

without reducing quality or security.

# **Trigger Conditions**

The workflow begins when:

* performance issues are detected  
* costs increase  
* scalability limits appear  
* reliability degrades  
* quality metrics decline  
* optimization opportunities are identified

Can also be triggered by:

* retrospective findings  
* benchmark failures  
* architecture reviews  
* monitoring alerts

Identify:

* slow components  
* expensive components  
* fragile systems  
* process inefficiencies  
* scaling limitations

Output:

Bottleneck Analysis Report

---

# **Stage 3 — Optimization Proposal**

Document:

* target improvement  
* expected impact  
* implementation cost  
* risk assessment  
* rollback plan

Output:

Optimization Proposal

---

# **Stage 4 — Architecture Review**

Validate:

* architectural integrity  
* maintainability  
* scalability  
* operational impact

Output:

Architecture Review Report

---

# **Stage 5 — Implementation**

Codex executes approved improvements.

Examples:

* code optimization  
* infrastructure tuning  
* database optimization  
* caching strategies  
* prompt optimization  
* workflow improvements

Output:

---

# **Stage 6 — Benchmark Validation**

Measure:

* before optimization  
* after optimization

* improvement is measurable  
* no regressions introduced

Output:

---

* functionality unchanged  
* no regressions

* no new vulnerabilities
